# VerifAI: AI Hallucination Detector

VerifAI is a comprehensive tool designed to automatically detect, explain, and correct false or made-up information (hallucinations) generated by AI models like ChatGPT, Gemini, and Claude. It acts as an auditor that works directly in your web browser. 

The project consists of three main parts:

1. Browser Extension: A Chrome extension that monitors the text generated by AI chatbots in real time. It adds a side panel and a floating button to the web page so you can verify claims without leaving your current tab.
2. Backend Server: A fast, Python-based backend that processes the text. It uses a consensus method by asking multiple advanced AI models (like Gemini, OpenAI, and Groq) to independently fact-check the claims. If they determine the text is a hallucination, the system generates a corrected version.
3. Analytics Dashboard: A visual interface built with Streamlit that lets you view statistics, track the kinds of errors the AI made, and understand the reasoning behind why a specific text was marked as a hallucination.

## How It Works

When you interact with an AI chatbot, the Chrome extension can verify the response. You click a button, and the extension sends the text to your local backend server. 

The backend then runs the text through an explainable AI pipeline. It checks the text against the multiple language models, highlights the exact parts of the text that are incorrect, categorizes the type of error, and suggests accurate information. The system also saves these results locally, so if you or someone else asks a similar question later, the verified answer loads instantly.

## Project Structure

- verifai-extension/: The code for the Chrome browser extension.
- api.py: The main programming interface (API) that runs the backend and handles requests from the browser extension.
- app.py: The dashboard application for viewing data and reports.
- src/: Contains the core intelligence and logic for the hallucination detection.
- data/: Stores background data and past cached results to speed up the verification process.

## Getting Started

To run the project locally on your machine, you need to start the backend API, the dashboard, and load the extension into your browser.

1. Ensure your API keys for the different language models are set up in the .env file.
2. Install the necessary Python packages using the requirements file.
3. Start the backend by running the api.py file.
4. Start the visual dashboard by running the app.py file.
5. Open Google Chrome, go to the extensions page, turn on Developer Mode, and load the verifai-extension folder as an unpacked extension.
